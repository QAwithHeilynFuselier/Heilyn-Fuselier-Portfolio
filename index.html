Selected work

Portfolio

Four deliberate lanes—a pure Java Unified Automation Framework core, a polyglot AI Data Bridge (Python, Java, JavaScript, TypeScript, SQLite), Go for performance probes, and n8n for workflows—show how execution stays on the JVM while integration spans languages with crisp boundaries.

Unified Automation Framework (Core)

Pure Java execution · AI self-healing

The Unified Automation Framework is intentionally a pure Java execution core: services, fixtures, test harnesses, and browser automation all run on the JVM using Playwright for Java so there is no secondary runtime in this lane. The emphasis is AI self-healing—selectors, waits, and recovery paths (including the AIHandler pattern) that adapt when the UI shifts so suites stay green without manual babysitting. The public reference implementation lives in Unified-Automation-Framework, with Maven modules and Docker for reproducible pipelines.

AI Data Bridge (Middleware)

Polyglot data engineering · AI middleware · ETL (XML → SQL)

This lane is the multi-language integration layer for polyglot data engineering: Python remains the primary engine for ETL that normalizes XML into SQLite with typed schemas and idempotent writes, while the lane still speaks Java, JavaScript, and TypeScript at the edges.

The bridge ingests Java and JavaScript artifacts from upstream tooling, applies Python intelligence for scoring and enrichment, and locks TypeScript contracts so payloads stay type-safe as they cross the stack—distinct from the pure Java execution core, which stays JVM-only in the first column.

It also folds in model evaluation for Project Terminus and Harbor, publishing versioned results the core can consume without re-implementing evaluation logic. The reference implementation lives in AI-Data-Bridge—decoupled from n8n orchestration, which remains in the observability column.

Go-Project (Infrastructure Sidecar)
